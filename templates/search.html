Under the hood

This is the code snippet in _macros.html


    <form class="b-search {{modifier}}" action="">
        <input class="js-search-value" type="text">
        <input class="js-search-submit js-search-trigger e-search-trigger material-icons" type="submit" value="search">
    </form>

This goes in scripts.js to make the class switching. Generally speaking, if you have that file included in _base, or is using the _base.html provided in this styleguide, you will be good to go.

This component also makes use of an auxiliary html element named #overlay, to enable the "outside click" to hide the search field, also included in the _base.html.


    var $overlay = $('#overlay');

    // Search Field Logic
    $('.js-search-trigger').click(function() {
        $(this).parent().addClass('m-active');
        $overlay.addClass('m-active');
    });
